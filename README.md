# Виконання роботи

## Використаний алгоритм

Для розв’язання пари взаємно двоїстих задач застосовано спільну симплекс-таблицю з подвійними мітками.

- Зліва розташовані базисні змінні прямої задачі `(y)` та двоїстої `(u)`
- Зверху — вільні змінні прямої `(x)` та двоїстої `(v)`
- Після кожного кроку МЖВ мітки обмінюються місцями одночасно для обох задач
- Розв’язок прямої задачі `(X)` зчитується з останнього стовпця
- Розв’язок двоїстої задачі `(U)` — з останнього рядка

Також використано правила побудови двоїстої задачі:

- якщо пряма задача на максимум — двоїста на мінімум;
- обмеження типу `≤` відповідають невід’ємним двоїстим змінним;
- невід’ємним змінним прямої задачі відповідають обмеження типу `≥` у двоїстій задачі.

# Побудова двоїстої задачі

## Пряма задача `(Z)`

```text
Z = x1 + 3x5 → max

2x1 + 2x2 + x3 + 2x4 + x5 ≤ 4
2x1 + x2 + 3x4 + 2x5 ≤ 4
3x1 + x3 + 2x4 + 5x5 ≤ 9

xj ≥ 0, j = 1..5
```

---

## Двоїста задача `(W)`

```text
W = 4u1 + 4u2 + 9u3 → min

2u1 + 2u2 + 3u3 ≥ 1
2u1 + u2 ≥ 0
u1 + u3 ≥ 0
2u1 + 3u2 + 2u3 ≥ 0
u1 + 2u2 + 5u3 ≥ 3

ui ≥ 0, i = 1..3
```

Для виконання роботи розроблено консольний застосунок мовою `C#`.

## Структура програми

### Клас `Program`

Відповідає за:

- взаємодію з користувачем;
- консольне меню;
- ініціалізацію готових задач.

### Клас `MJV`

Містить:

- реалізацію модифікованих жорданових виключень;
- побудову двоїстої задачі;
- пошук опорного розв’язку;
- пошук оптимального розв’язку.
